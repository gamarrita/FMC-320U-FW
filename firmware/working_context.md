# WORKING_CONTEXT.md

## Purpose

This file is the live working context for current or recent firmware tasks.

It exists to preserve continuity across sessions.
It is mutable by design.

This file is not a source of truth for repository rules.

Do not use this file to define permanent rules for:

- workflow
- style
- naming
- architecture
- folder responsibilities

Those belong in stable documents such as:

- `AGENT_ENTRY.md`
- `STYLE.md`
- `AGENTS.md`
- `workflows/README.md`

---

## Workflow reference

All workflow stages are defined under:

- `workflows/README.md`

Use stage-based execution when appropriate:
- analysis
- plan
- implementation
- comment pass
- validation

This file should not redefine workflow behavior.
