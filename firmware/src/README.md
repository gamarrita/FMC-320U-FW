# src

## Intent

Contain firmware source authored by this project.

CubeMX-generated code, vendor drivers, middleware, and the `.ioc` project stay
at repository root in the layout expected by CubeMX. This keeps CubeMX usable
for changing RTOS settings, enabling middleware, adding peripherals, and
regenerating initialization code.

## Layout

- `apps/`: selectable firmware application profiles.
- `bsp/`: board and device-facing hardware details.
- `port/`: MCU, toolchain, and RTOS/base-runtime adaptation.
- `product/`: FMC-320U product semantics and behavior.
- `services/`: reusable technical services.
- `libs/`: small reusable support libraries that are not product semantics.

## Rule

If the code is firmware source written by this project and is not generated by
CubeMX, it belongs here.
