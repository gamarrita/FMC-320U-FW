# Safety Pass

## Role

Focused methodological pass for safety-like checks and related hardening work.

## When to use

Use it when safety-oriented review is the explicit task.

## What it does

- checks safety-relevant concerns in the requested scope
- documents or reports safety-like issues found during the pass
- supports later hardening work

## What it does NOT do

- does not create or replace `WORKING_CONTEXT.md`
- does not become the default concern of all stages
- does not justify unrelated redesign by default

## Working context

- requires existing `WORKING_CONTEXT.md`: Yes
- can modify it: No
- creates it: No

## Requires active file (log + contract)

No

## Output

A focused safety-oriented review or hardening pass over the requested scope.
