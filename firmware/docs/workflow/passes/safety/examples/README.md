# Safety Pass Examples

## Purpose

These files are illustrative examples for the safety pass.

Use them to understand:
- how a safety review can be reported
- how a small hardening result can be framed
- how to keep scope small and explicit

## Important Limit

These examples are not policy.

If an example conflicts with:
- `WORKING_CONTEXT.md`
- `STYLE.md`
- `docs/workflow/passes/README.md`
- `docs/workflow/passes/safety/CONTRACT.md`

the example loses.

## Files

- `review_example.md`
  Example of a focused safety review output.
- `hardening_example.md`
  Example of a small hardening outcome that stays inside safety-pass scope.
