# Refactor Pass

## Role

Bootstrap methodological pass for starting or reframing refactor work.

## When to use

Use it when a refactor becomes a real line of work and needs explicit framing.

## What it does

- defines or reframes the current refactor focus
- can establish or update `WORKING_CONTEXT.md`
- can leave an active working document aligned with that refactor

## What it does NOT do

- does not replace stable repository references
- does not justify broad redesign by default
- does not turn every local cleanup into a refactor track

## Working context

- requires existing `WORKING_CONTEXT.md`: No
- can modify it: Yes
- creates it: Yes

## Requires active file (log + contract)

Yes

## Output

A reframed or newly established refactor context with clear current focus.
