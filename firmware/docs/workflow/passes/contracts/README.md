# Contracts Pass

## Role

Focused methodological pass for stronger contract clarity.

## When to use

Use it when APIs, boundaries, or behavioral contracts need explicit cleanup.

## What it does

- clarifies contracts
- aligns public behavior with documented expectations
- tightens boundary semantics where that is the explicit task

## What it does NOT do

- does not create or replace `WORKING_CONTEXT.md`
- does not become a broad redesign pass
- does not justify unrelated implementation work

## Working context

- requires existing `WORKING_CONTEXT.md`: Yes
- can modify it: No
- creates it: No

## Requires active file (log + contract)

No

## Output

A focused contract-cleanup pass over the requested scope.
