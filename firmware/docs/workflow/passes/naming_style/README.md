# Naming And Style Pass

## Role

Focused methodological pass for stronger naming and style cleanup.

## When to use

Use it after the main behavior is stable and naming or style is the explicit task.

## What it does

- improves naming consistency
- improves style consistency
- tightens readability and local convention adherence

## What it does NOT do

- does not create or replace `WORKING_CONTEXT.md`
- does not become the default concern of every stage
- does not justify unrelated behavior changes

## Working context

- requires existing `WORKING_CONTEXT.md`: Yes
- can modify it: No
- creates it: No

## Requires active file (log + contract)

No

## Output

A focused naming and style cleanup pass over the requested scope.
