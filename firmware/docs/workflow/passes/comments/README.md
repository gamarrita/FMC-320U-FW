# Comments Pass

## Role

Focused methodological pass for comment quality and consistency.

## When to use

Use it after implementation or validation when comments are the explicit task.

## What it does

- improves public and private comments
- removes contradictory or noisy comments
- aligns comment style with repository expectations

## What it does NOT do

- does not create or replace `WORKING_CONTEXT.md`
- does not become the main workflow entrypoint
- does not justify unrelated implementation changes

## Working context

- requires existing `WORKING_CONTEXT.md`: Yes
- can modify it: No
- creates it: No

## Requires active file (log + contract)

No

## Output

A focused comment cleanup pass over the requested scope.
