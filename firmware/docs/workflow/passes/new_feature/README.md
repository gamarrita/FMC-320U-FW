# New Feature Pass

## Role

Bootstrap methodological pass for starting a new feature track.

## When to use

Use it when a new feature becomes a real line of work and needs explicit framing.

## What it does

- defines the current feature focus
- can establish or update `WORKING_CONTEXT.md`
- can leave an active working document aligned with that feature

## What it does NOT do

- does not replace stable repository references
- does not justify speculative feature expansion
- does not become a general project planning layer

## Working context

- requires existing `WORKING_CONTEXT.md`: No
- can modify it: Yes
- creates it: Yes

## Requires active file (log + contract)

Yes

## Output

A newly established or reframed feature context with clear current focus.
